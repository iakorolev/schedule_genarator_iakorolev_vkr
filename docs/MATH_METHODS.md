# Математические методы в алгоритме распределения

В текущей версии проекта в ядро распределения добавлены формализованные методы, которые можно использовать в описательной и аналитической части ВКР.

## 1. Матрица компетенций преподавателей

На основе файла РУН строится матрица компетенций
\[
A = \{a_{ijk}\},
\]
где:
- \(i\) — преподаватель,
- \(j\) — дисциплина,
- \(k\) — вид занятия.

Вес компетенции определяется по сумме часов и числу групп, с которыми преподаватель связан в РУН.

Практически матрица выгружается в файлы:
- `teacher_competency_matrix.xlsx`
- `teacher_competency_pivot.xlsx`

## 2. Нормированные критерии выбора кандидата

Для каждого кандидата на слот рассчитываются критерии в диапазоне \([0,1]\):
- `subject_fit` — предметная пригодность,
- `day_fit` — присутствие преподавателя в нужный день,
- `load_fit` — соответствие по остаточной нагрузке,
- `continuity_fit` — согласованность с уже сделанными назначениями,
- `hint_fit` — совпадение с teacher hint из расписания,
- `new_day_penalty` — штраф за открытие нового дня.

## 3. Метод взвешенной суммы

Для абсолютной оценки кандидата используется взвешенная свёртка критериев:
\[
S_{wsm}(x)=\sum_{m=1}^{M} w_m c_m(x),
\]
где \(w_m\) — вес критерия, \(c_m(x)\) — нормированное значение критерия.

## 4. Метод TOPSIS

Для ранжирования нескольких кандидатов внутри одного слота используется TOPSIS:
1. строится матрица критериев,
2. выполняется векторная нормализация,
3. применяется взвешивание,
4. считаются расстояния до идеального и анти-идеального решения,
5. вычисляется коэффициент близости:
\[
C_i = \frac{D_i^-}{D_i^+ + D_i^-}.
\]

Чем больше \(C_i\), тем кандидат ближе к оптимальному решению.

## 5. Гибридная итоговая оценка

В программе используется гибрид:
- экспертная эвристическая оценка,
- взвешенная сумма,
- TOPSIS.

Это позволяет сохранить рабочий функционал системы и одновременно формализовать алгоритм для ВКР.
