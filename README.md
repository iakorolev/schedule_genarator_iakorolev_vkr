## О проекте

Данный проект представляет собой прототип системы автоматического формирования преподавательского расписания кафедры на основе двух внешних Excel-источников:

- **РУН / УН** — файл учебной нагрузки преподавателей;
- **общее расписание** — файл с расписанием занятий по группам.

Практический смысл проекта состоит в том, что преподавательское расписание ранее формировалось **вручную** на основе разнородных табличных данных. Разрабатываемая система автоматизирует этот процесс: извлекает данные из Excel-файлов, нормализует дисциплины, виды занятий и группы, выделяет атомарные слоты расписания, выполняет назначения по строгим правилам, а для неоднозначных случаев использует эвристическое ранжирование и ручную корректировку через веб-интерфейс.

> **Идея проекта:** не просто “назначить преподавателя”, а автоматизировать реальный кафедральный процесс, включающий извлечение, согласование и интерпретацию данных.

---

## Ключевой функционал

- загрузка файлов **РУН** и **расписания** через веб-интерфейс;
- разбор и нормализация Excel-источников;
- выделение кафедральных слотов из общего расписания;
- автоматические назначения по точным совпадениям и строгим правилам;
- эвристическое дораспределение неоднозначных случаев;
- фиксация конфликтов и низкоуверенных назначений;
- просмотр нераспределённых слотов;
- ручной выбор преподавателя для проблемных случаев;
- пересборка черновика с учётом ручных правил;
- экспорт итогового преподавательского расписания в Excel.

## Входные данные

### 1. Файл РУН / УН
Используется как источник учебной нагрузки преподавателей кафедры.

Ожидается наличие листа:

- `УН сводная`

Из файла извлекаются:
- ФИО преподавателя;
- дисциплина;
- вид занятия;
- группы;
- часы;
- семестр;
- дополнительные атрибуты нагрузки.

### 2. Файл общего расписания
Используется как источник сетки занятий по группам.

Ожидается наличие листа:

- `ФМиЕН`

Из файла извлекаются:
- день недели;
- номер пары;
- время;
- группа;
- дисциплина;
- вид занятия;
- аудитория / дистанционный формат;
- возможные подсказки по преподавателю.

## Выходные данные

В зависимости от версии пайплайна система может формировать:

- `schedule_with_teachers.xlsx` — расписание с назначенными преподавателями;
- `timetable_by_teachers.xlsx` — итоговое преподавательское расписание;
- `unmatched_slots.xlsx` — нераспределённые или спорные слоты;
- `lowconf_matches.xlsx` — низкоуверенные назначения;
- `teacher_conflicts.xlsx` — конфликтные назначения;
- `run_atoms.xlsx` — атомарные элементы нагрузки;
- `schedule_atoms.xlsx` — атомарные элементы расписания;
- `locked_assignments.xlsx` — назначения по строгим правилам;
- `allocated_assignments.xlsx` — назначения эвристического этапа;
- `candidate_scores.xlsx` — кандидаты и их оценки;
- `teacher_load_summary.xlsx` — сводка по нагрузке преподавателей.

## Пример результатов тестового прогона

На рабочих входных файлах система показала следующие результаты:

| Показатель | Значение |
|---|---:|
| Общее число кафедральных слотов после фильтрации | **222** |
| Успешно распределено | **209** |
| Осталось нераспределёнными | **13** |
| Назначено по строгим правилам | **164** |
| Назначено автоматически эвристическим этапом | **45** |
| Исключено как внешние / некафедральные слоты | **1051** |
| Зафиксировано конфликтов | **3** |
| Низкоуверенных назначений | **3** |

Фрагмент консольного вывода:

```text
run загружен: C:\Users\User\Downloads\...\run.xlsx
sched загружен: C:\Users\User\Downloads\...\sched.xlsx
строю черновик...
ok
{
  "auto_allocated": 45,
  "conflicts": 3,
  "external_excluded": 1051,
  "locked": 164,
  "lowconf": 3,
  "matched": 209,
  "total_slots": 222,
  "unmatched": 13
}
```

### Как интерпретировать результат

- **222 кафедральных слота после фильтрации** — занятия, отнесённые к целевой кафедральной части расписания;
- **209 успешно распределено** — число слотов, для которых преподаватель был назначен автоматически;
- **13 нераспределённых** — случаи, оставленные на дополнительную проверку или ручную корректировку;
- **164 назначено по строгим правилам** — слоты, распределённые по точным совпадениям и формализованным условиям;
- **45 назначено эвристически** — случаи, распределённые с помощью системы оценивания кандидатов;
- **1051 внешний / некафедральный слот** — занятия, сознательно исключённые из текущей постановки;
- **3 конфликта** — ситуации, требующие дополнительной проверки;
- **3 низкоуверенных назначения** — случаи, которые рекомендуется просмотреть вручную.

> Важно: часть нераспределённых слотов не является ошибкой системы. В некоторых случаях для занятия существуют несколько равноценных преподавателей-кандидатов, и прототип сознательно не делает произвольный выбор без дополнительных оснований.

## Почему этот проект не сводится к одному алгоритму

Система решает не только задачу выбора преподавателя, но и задачу интерпретации исходных табличных данных. Excel-файлы неоднородны по структуре: в них встречаются объединённые ячейки, подгруппы, потоки, смешанные обозначения видов занятий, а часть сведений о преподавателе задаётся косвенно. Поэтому в проекте используется не один «универсальный» алгоритм, а последовательность этапов:

- парсинг Excel-источников;
- нормализация дисциплин, групп и видов занятий;
- формирование атомарной внутренней модели;
- назначения по строгим правилам;
- эвристическое ранжирование кандидатов;
- ручная корректировка неоднозначных случаев.

Именно такая гибридная схема делает прототип применимым к реальному кафедральному процессу.

- система ориентирована на конкретные форматы входных Excel-файлов;
