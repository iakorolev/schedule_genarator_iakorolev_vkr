# Генератор кафедрального преподавательского расписания

## Описание проекта

Данный проект представляет собой прототип системы автоматического формирования преподавательского расписания кафедры на основе двух внешних Excel-источников:

- **РУН / УН** — файл учебной нагрузки преподавателей;
- **общее расписание** — файл с расписанием занятий по группам.

Цель проекта — автоматизировать построение кафедрального преподавательского расписания, которое ранее формировалось вручную. Система извлекает данные из Excel-файлов, нормализует дисциплины, виды занятий и группы, выделяет атомарные слоты расписания, выполняет назначения по строгим правилам и дораспределяет оставшиеся случаи эвристическим методом. Для спорных ситуаций предусмотрен веб-интерфейс с возможностью ручной корректировки.

Проект разработан в рамках НИР / ВКР и ориентирован на решение прикладной задачи кафедрального уровня.

---

## Ключевой функционал

Реализованный прототип поддерживает следующий функционал:

- загрузка файлов **РУН** и **расписания** через веб-интерфейс;
- разбор и нормализация Excel-источников;
- выделение кафедральных слотов из общего расписания;
- автоматические назначения по точным совпадениям и строгим правилам;
- эвристическое дораспределение неоднозначных случаев;
- фиксация конфликтов и низкоуверенных назначений;
- просмотр нераспределённых слотов;
- ручной выбор преподавателя для проблемных случаев;
- пересборка черновика с учётом ручных правил;
- экспорт итогового преподавательского расписания в Excel.

## Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/iakorolev/schedule_genarator_iakorolev_vkr.git
cd schedule_genarator_iakorolev_vkr
```

### 2. Создание виртуального окружения

**Windows:**

```bash
python -m venv .venv
.venv\Scripts\activate
```

**Linux / macOS:**

```bash
python -m venv .venv
source .venv/bin/activate
```

```bash

```

Если файла `requirements.txt` ещё нет, зависимости можно установить вручную:

```bash
pip install flask pandas openpyxl
```

## Входные данные

### 1. Файл РУН / УН

Используется как источник учебной нагрузки преподавателей кафедры.

Ожидается наличие листа:

- `УН сводная`

Из файла извлекаются:

- ФИО преподавателя;
- дисциплина;
- вид занятия;
- группы;
- часы;
- семестр;
- дополнительные атрибуты нагрузки.

### 2. Файл общего расписания

Используется как источник сетки занятий по группам.

Ожидается наличие листа:

- `ФМиЕН`

Из файла извлекаются:

- день недели;
- номер пары;
- время;
- группа;
- дисциплина;
- вид занятия;
- аудитория / дистанционный формат;
- возможные подсказки по преподавателю.

## Выходные данные

В результате работы прототип формирует набор Excel-файлов. В зависимости от текущей версии пайплайна набор может включать:

- `schedule_with_teachers.xlsx` — расписание с назначенными преподавателями;
- `timetable_by_teachers.xlsx` — итоговое преподавательское расписание;
- `unmatched_slots.xlsx` — нераспределённые или спорные слоты;
- `lowconf_matches.xlsx` — низкоуверенные назначения;
- `teacher_conflicts.xlsx` — конфликтные назначения;
- `run_atoms.xlsx` — атомарные элементы нагрузки;
- `schedule_atoms.xlsx` — атомарные элементы расписания;
- `locked_assignments.xlsx` — назначения по строгим правилам;
- `allocated_assignments.xlsx` — назначения эвристического этапа;
- `candidate_scores.xlsx` — кандидаты и их оценки;
- `teacher_load_summary.xlsx` — сводка по нагрузке преподавателей.

## Пример результатов тестового прогона

На рабочих входных файлах система показала следующие результаты:

- **общее число кафедральных слотов после фильтрации:** 222;
- **успешно распределено:** 213;
- **осталось нераспределёнными:** 9;
- **назначено по строгим правилам:** 172;
- **назначено автоматически эвристическим этапом:** 41;
- **исключено как внешние / некафедральные слоты:** 1051;
- **зафиксировано конфликтов:** 6;
- **низкоуверенных назначений:** 3.

Фрагмент консольного вывода:

```text
run загружен: C:\Users\User\Downloads\full-version-auto-rules-v2\dep_patch\webapp\data\sessions\20260317_162241\run.xlsx
sched загружен: C:\Users\User\Downloads\full-version-auto-rules-v2\dep_patch\webapp\data\sessions\20260317_162241\sched.xlsx
строю черновик...
ok
{
  "auto_allocated": 41,
  "conflicts": 6,
  "external_excluded": 1051,
  "locked": 172,
  "lowconf": 3,
  "matched": 213,
  "total_slots": 222,
  "unmatched": 9
}
```

### Интерпретация результатов

- **222 кафедральных слота после фильтрации** — занятия, которые система отнесла именно к целевой кафедральной части расписания;
- **213 успешно распределено** — число слотов, для которых преподаватель был назначен автоматически;
- **9 нераспределённых** — случаи, оставленные на дополнительную проверку или ручную корректировку;
- **172 назначено по строгим правилам** — слоты, распределённые по точным совпадениям и формализованным условиям;
- **41 назначено эвристически** — случаи, распределённые с помощью системы оценивания кандидатов;
- **1051 внешний / некафедральный слот** — занятия, исключённые из текущей кафедральной постановки;
- **6 конфликтов** — ситуации, требующие дополнительной проверки;
- **3 низкоуверенных назначения** — случаи, в которых система назначила преподавателя, но отмечает решение как требующее контроля.

Важно отметить, что часть нераспределённых слотов не является ошибкой системы. В некоторых случаях для занятия существуют несколько равноценных преподавателей-кандидатов: они соответствуют дисциплине, не имеют временного конфликта и допустимы по нагрузке. В такой ситуации прототип сознательно не делает произвольный выбор, а передаёт случай на экспертную проверку.

## Реализованный функционал в сравнении с запланированным

### Что было запланировано

На этапе постановки задачи предполагалось создать систему, которая:

- принимает РУН и общее расписание;
- автоматически формирует кафедральное преподавательское расписание;
- позволяет учитывать особенности Excel-файлов и неоднозначность исходных данных;
- сохраняет возможность ручного вмешательства в спорных случаях;
- формирует итоговый файл, пригодный для практического использования.

### Что реализовано в прототипе

В текущей версии реализованы:

- загрузка входных Excel-файлов;
- разбор и нормализация данных;
- автоматическое распределение значительной части слотов;
- обработка teacher hints и точных совпадений;
- эвристический выбор преподавателя в неоднозначных случаях;
- интерфейс для просмотра проблемных мест;
- механизм ручных правил и повторной сборки;
- экспорт результатов в Excel-файлы.

### Что реализовано частично / остаётся на развитие

- полное автоматическое воспроизведение исторического многостраничного итогового формата кафедры;
- более точная локальная оптимизация конфликтных случаев;
- расширение набора правил нормализации дисциплин и подгрупп;
- усиление автоматической проверки качества результатов.

Таким образом, прототип уже реализует ключевой функционал системы и позволяет использовать его как рабочий инструмент для подготовки преподавательского расписания, хотя часть функций ещё находится на этапе развития.

## Новизна и практическая ценность

Новизна проекта состоит не в применении одного универсального алгоритма, а в построении гибридной системы, которая сочетает:

- интеллектуальный разбор полуструктурированных Excel-источников;
- нормализацию предметных сущностей;
- назначения по строгим правилам;
- эвристическое ранжирование кандидатов;
- ручное экспертное вмешательство для неоднозначных случаев.

Практическая ценность прототипа заключается в том, что он автоматизирует реальную кафедральную задачу, ранее выполнявшуюся вручную, и позволяет существенно сократить объём рутинной работы при сохранении управляемости результата.

## Ограничения текущей версии

- система ориентирована на конкретные форматы входных Excel-файлов;
- корректность работы зависит от структуры листов `УН сводная` и `ФМиЕН`;
- часть спорных случаев требует ручного подтверждения;
- итоговый выходной формат пока ближе к техническому преподавательскому представлению, чем к полностью воспроизведённому историческому шаблону кафедры.
